# Kopach3D — Воксельный редактор на Marching Cubes

Редактор трёхмерного ландшафта, где можно добавлять и удалять блоки разной формы: сферы, линии, плоскости. Ландшафт строится алгоритмом Marching Cubes, который превращает воксельное поле плотности в гладкую полигональную поверхность.

## Возможности

- Передвижение по миру от первого лица (WASD/QE + мышь)
- Добавление блоков: сфера с плавным градиентом (приближена к шару), линия вдоль оси, плоскость
- Удаление блоков (левая кнопка мыши)
- Превью кисти (wireframe-сфера меняет цвет при действии)
- Сохранение и загрузка карты (бинарный файл map.bin)
- Настройка цвета ландшафта, освещения, размера кисти через панель ImGui
- Режим wireframe (F1) — видно сетку треугольников
- Оконное приложение без консоли

## Что нужно для компиляции (Windows)

Установите:

1. **Visual Studio 2026** (Community Edition бесплатна) — выберите компонент «Разработка классических приложений на C++» при установке
2. **CMake** (>= 3.20) — скачайте с [cmake.org](https://cmake.org/download/), при установке выберите «Add CMake to system PATH»
3. **Git** — скачайте с [git-scm.com](https://git-scm.com/download/win)

## Как собрать (Windows, командная строка)

Откройте командную строку (cmd или PowerShell) и выполните:

```
cd путь\к\папке\проекта
mkdir build
cd build
cmake ..
cmake --build . --config Release
```

После успешной сборки exe-файл будет лежать в `build\bin\Release\Kopach3D.exe`.

Первая сборка может занять несколько минут — CMake скачает зависимости (GLFW, GLM, ImGui) из GitHub и скомпилирует их.

## Системные требования (для запуска готового exe)

- **Windows 10/11** (x64)
- **OpenGL 4.6** — видеокарта с поддержкой OpenGL 4.6 (любая современная Nvidia/AMD/Intel)
- **Visual C++ Redistributable 2015–2022 (x64)** — если при запуске появляется ошибка `VCRUNTIME140.dll не найден`, скачайте и установите: [https://aka.ms/vs/17/release/vc_redist.x64.exe](https://aka.ms/vs/17/release/vc_redist.x64.exe)

## Как запустить

Распакуйте архив в любую папку и запустите `Kopach3D.exe`. Рядом с ним должна лежать папка `assets\shaders\` с шейдерами.

## Управление

| Клавиша | Действие |
|---------|----------|
| WASD | Движение по горизонтали |
| Q / E | Вверх / вниз |
| Мышь | Поворот камеры |
| TAB | Переключить захват мыши (камера ↔ интерфейс) |
| ЛКМ | Удалить блок (сделать воздухом) |
| ПКМ | Добавить блок |
| Колёсико | Приблизить / отдалить (изменить FOV) |
| F1 | Включить / выключить wireframe |
| ESC | Выход |

## Параметры кисти (в панели ImGui)

| Режим | Описание |
|-------|----------|
| Sphere (smooth) | Сфера с плавным градиентом плотности — гладкая поверхность |
| Axis X / Y / Z | Линия вокселей вдоль выбранной оси |
| Plane XZ / XY / YZ | Плоскость, параллельная двум выбранным осям |

Радиус кисти регулируется ползунком (1–10 вокселей).

## Как собрать релизную версию без лишних файлов

1. Удалите папки `build/` и `build-release/` (если есть)
2. Создайте чистую папку для сборки и соберите проект:

```
mkdir build
cd build
cmake ..
cmake --build . --config Release
```

3. Готовый релиз лежит в `build\bin\Release\`:
   - `Kopach3D.exe`
   - `assets\` (шейдеры)

Эти два элемента можно упаковать в zip и раздавать — больше ничего не нужно.

## Структура проекта

```
├── CMakeLists.txt          — система сборки CMake
├── README.md               — этот файл
├── assets/
│   └── shaders/
│       ├── terrain.vert    — вершинный шейдер
│       └── terrain.frag    — фрагментный шейдер
├── include/
│   ├── engine/             — заголовки движка (App, Camera, Shader, Mesh)
│   ├── terrain/            — заголовки ландшафта (VoxelWorld, MarchingCubes)
│   ├── glad/               — заголовки OpenGL-загрузчика
│   └── KHR/                — заголовки Khronos (OpenGL)
├── libs/
│   └── glad/               — исходники GLAD (загрузчик OpenGL 4.6)
└── src/
    ├── main.cpp            — точка входа
    ├── engine/             — реализация движка
    └── terrain/            — реализация ландшафта и Marching Cubes
```

## Зачем Git для компиляции

CMake использует модуль FetchContent, который скачивает библиотеки (GLFW, GLM, ImGui) прямо с GitHub во время первой сборки. Поэтому Git должен быть установлен и доступен в командной строке.
